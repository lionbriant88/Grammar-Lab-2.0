## 0. 架构强制要求 (Architectural Mandates)

### 0.1 移除 PP_RULES ✅

**理由**: PP 内部包含 NP，扩展 PP 就是扩展其内部 NP，不需要独立规则。

```python
# ❌ 错误 - 不要创建 PP_RULES
PP_RULES = ["expand_np"]

# ✅ 正确 - PP 无独立规则，点击 PP 自动聚焦其 children NP
# 例: "in the park" → PP(children=[NP(park)])
# 扩展逻辑: 点击 PP → 自动选中 NP(park) → 应用 NP_RULES
```

**实现**: UI 层在点击 PP 节点时，自动查找 `phrase.children_ids` 中的 NP 并聚焦。

### 0.2 移除 PARTICIPLE_RULES ✅

**理由**: Participle 本质是 VP，应复用 VP_RULES，通过 `verb_form` 特征区分。

```python
# ❌ 错误 - 不要创建新的 PhraseType.PARTICIPLE
PhraseType = Literal["NP", "VP", "PP", "PARTICIPLE", ...]

# ✅ 正确 - Participle 是 VP，通过 features 区分
PhraseNode(
    type="VP",
    text="running quickly",
    features={
        "verb_form": "present_participle",  # 新增特征
        "tense": None,
        ...
    }
)
```

**verb_form 枚举**:
- `finite` - 限定动词 (has been working)
- `present_participle` - 现在分词 (running)
- `past_participle` - 过去分词 (covered)
- `infinitive` - 不定式 (to help)
- `gerund` - 动名词 (swimming is fun)

**规则复用**: `VP_RULES` 根据 `verb_form` 动态过滤候选项。

### 0.3 移除跨 VP 时态一致性检查 ✅

**理由**: 跨句子/从句的时态一致性需要语义理解，属于 M4 AI Validator 范畴。

```python
# ❌ 错误 - 不要实现跨 VP 时态比较

    vps = [p for p in phrases if p.syntactic_role == "predicate"]
    if len(vps) >= 2:
        # 比较 vps[0].tense vs vps[1].tense ← 语义级逻辑，规则引擎无法判断
        ...

# ✅ 正确 - 实现单 VP 内部的助动词链一致性
def validate_aux_chain_integrity(phrases):
    """检查单个 VP 内部的 aux_chain 是否合法。
    
    示例:
    - ✅ has been working (aux_chain=[has,been], tense=present_perfect_progressive)
    - ❌ have been working (主语 He 第三人称单数，have 错误)
    - ❌ has working (缺少 been)
    - ❌ is worked (past_participle 需要 have，不是 is)
```

**实现**: 检查 `VP.features["aux_chain"]` 与 `VP.features["tense"]` 的组合是否合法。

## 3. Phase 2: PhraseSegmenter 增强

### 3.1 segment_benepar() 实现

**文件**: `backend/grammar_engine/phrase_segmenter.py`

**流程**:
1. `parser.parse(doc.sents)` → nltk.Tree
2. DFS 遍历树，每个非终结符 → PhraseNode:
   - `(NP ...)` → `type="NP"`
   - `(VP ...)` → `type="VP"`, 填充 `verb_form`
   - `(PP ...)` → `type="PP"`
   - `(ADJP ...)` → `type="ADJP"`
   - `(ADVP ...)` → `type="ADVP"`
   - `(S ...)` / `(SBAR ...)` → `type="CLAUSE"`
3. Parent-Child 挂载直接来自树结构
4. 填充特征槽:
   - NP: `number` / `person`
   - VP: `verb_form` / `tense` / `aux_chain` / `modal` / `aspect`
5. 填充 M3b 新增字段:
   - `head_word` = `head_token_text`
   - `role` = `syntactic_role`
   - `modifiers` = 从 `children_ids` 中提取修饰性短语的 id 列表

### 3.2 verb_form 识别

```python
def extract_verb_form(vp_tree, doc) -> str:
    """从 Benepar VP 树提取 verb_form。
    
    规则:
    - VP 含 MD (modal) → finite
    - VP 含 VBZ/VBP/VBD + 非 aux → finite
    - VP 含 VBG 且无 aux 前置 → present_participle
    - VP 含 VBN 且无 aux 前置 → past_participle
    - VP 含 TO → infinitive
    - VP 首位是 VBG 且作主语/宾语 → gerund
    """
```

## 5. Phase 4: Validator - Aux Chain Integrity

### 5.1 实现 validate_aux_chain_integrity()

**文件**: `backend/grammar_engine/expansion_validator.py`

```python
def validate_aux_chain_integrity(sentence: str, doc, phrases: list[PhraseNode]) -> ValidationReport:
    """检查每个 VP 内部的助动词链是否合法。
    
    检查规则:
    1. present_perfect → aux_chain 必须含 "have/has"
    2. present_perfect_progressive → aux_chain 必须含 "have/has" + "been"
    3. past_perfect → aux_chain 必须含 "had"
    4. passive → 必须有 "be" + past_participle
    5. 主语 3sg + simple_present → 动词必须带 -s (已在 subject_verb_agreement 检查)
    6. modal + main_verb → modal 后必须是原形动词
    
    示例错误:
    - "He has working." → ERROR: present_perfect 需要 past_participle，不是 present_participle
    - "He is worked." → ERROR: "is + worked" 应该是 passive，但 worked 的语义不支持被动
    - "He have been working." → ERROR: 主语 3sg，have 应为 has (已被 subject_verb_agreement 捕获)
    """
    errors = []
    warnings = []
    
    for phrase in phrases:
        if phrase.type != "VP" or phrase.syntactic_role != "predicate":
            continue
        
        tense = phrase.features.get("tense")
        aux_chain = phrase.features.get("aux_chain", [])
        verb_form = phrase.features.get("verb_form", "finite")
        
        # 检查 1: present_perfect 必须有 have/has
        if tense == "present_perfect":
            if not any(aux in ["have", "has"] for aux in aux_chain):
                errors.append(f"VP '{phrase.text}' 标记为 present_perfect，但 aux_chain 缺少 have/has")
        
        # 检查 2: present_perfect_progressive 必须有 have/has + been
        if tense == "present_perfect_progressive":
            if not ("has" in aux_chain or "have" in aux_chain) or "been" not in aux_chain:
                errors.append(f"VP '{phrase.text}' 标记为 present_perfect_progressive，但 aux_chain 不完整")
        
        # 检查 3: past_perfect 必须有 had
        if tense == "past_perfect":
            if "had" not in aux_chain:
                errors.append(f"VP '{phrase.text}' 标记为 past_perfect，但 aux_chain 缺少 had")
        
        # 更多规则...
    
    severity = "ERROR" if errors else ("WARNING" if warnings else "PASS")
    return ValidationReport(
        severity=severity,
        is_valid=(severity == "PASS"),
        errors=errors,
        warnings=warnings,
        infos=[],
        auto_corrections=[]
    )
```

### 6.2 短语关系信息面板 (新增)

**文件**: `src/renderer/components/expand/PhraseRelationPanel.tsx`

```tsx
interface PhraseRelationPanelProps {
  phrase: PhraseNodeInfo | null;
  allPhrases: PhraseNodeInfo[];
  darkMode: boolean;
}

export default function PhraseRelationPanel({ phrase, allPhrases, darkMode }: PhraseRelationPanelProps) {
  if (!phrase) return <div>未选中短语</div>;
  
  // 找父节点
  const parent = allPhrases.find(p => p.id === phrase.parent_id);
  
  // 找子节点
  const children = allPhrases.filter(p => phrase.children_ids.includes(p.id));
  
  // 找修饰语
  const modifiers = allPhrases.filter(p => phrase.modifiers?.includes(p.id));
  
  return (
    <div className="space-y-3">
      <h4 className="font-bold">当前短语: {phrase.text}</h4>
      
      <div>
        <span className="text-xs opacity-60">语法角色:</span>
        <span className="ml-2">{phrase.role}</span>
      </div>
      
      <div>
        <span className="text-xs opacity-60">中心词:</span>
        <span className="ml-2">{phrase.head_word}</span>
      </div>
      
      {modifiers.length > 0 && (
        <div>
          <span className="text-xs opacity-60">修饰语:</span>
          <div className="ml-2 space-y-1">
            {modifiers.map(m => <div key={m.id}>{m.text}</div>)}
          </div>
        </div>
      )}
      
      {children.length > 0 && (
        <div>
          <span className="text-xs opacity-60">包含:</span>
          <div className="ml-2 space-y-1">
            {children.map(c => <div key={c.id}>{c.type}({c.text})</div>)}
          </div>
        </div>
      )}
      
      {parent && (
        <div>
          <span className="text-xs opacity-60">父节点:</span>
          <span className="ml-2">{parent.type}({parent.text})</span>
        </div>
      )}
    </div>
  );
}
```
